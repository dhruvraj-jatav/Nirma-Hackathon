The application first defines a set of colors to be detected in the input image, and creates a mask for each color range using the OpenCV library. It then counts the number of pixels that match each color mask, and adds the color name and count to a list if there are any matching pixels.
The script then determines the color of the water by checking for blue or green pixels in the detected colors list. If there are blue pixels, the water is considered clear. If there are green pixels, the water is considered to have high organic content. If neither color is detected, the water is considered white.
The script also determines the color of the sediment by checking for brown pixels in the detected colors list. If brown pixels are detected, the sediment is considered brown. Otherwise, it is considered not detected.
The script then calculates the turbidity of the water by applying edge detection using the Canny algorithm in OpenCV to the grayscale image. The edges are dilated to fill gaps and improve object detection, and contours are found using the find Contours function in OpenCV. The mean value of the edges is used to calculate the turbidity.
The script then draws the detected contours on the input image using the draw Contours function in OpenCV, and saves the image with contours. Finally, it converts the image to base64 for display in the HTML template, and returns the results to the template.
This is a Python script that creates a web application using the Flask web framework. The application takes an image as input and analyzes it to determine the color of the water and sediment, as well as the turbidity of the water.
